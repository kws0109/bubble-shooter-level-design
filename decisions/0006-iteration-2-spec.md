# ADR 0006 — Iteration 2: 레벨 진행, 색 swap 슈터, 좌우 대칭

- **상태**: 확정
- **일자**: 2026-05-06
- **선행 결정**: [ADR 0005](0005-multicollinearity-finding.md)

## 맥락

Iteration 1은 *난이도 측정 프레임*을 검증했다. Iteration 2는 그 위에 **실제 라이브 게임 메커닉**을 얹는다. 디자이너 피드백 다섯 가지:

1. 레벨 진행에 따라 색 점진 도입 (10레벨 단위, 50 이후 고정).
2. 클리어 조건 = 모든 버블 삭제. *(Iteration 1에서 이미 구현)*
3. 부유 버블 자동 삭제. *(Iteration 1에서 이미 구현 — `floating_cells()`)*
4. (#2와 동일)
5. 발사 버블이 *2색 가변* (color swap, Bubble Pop Origin 표준).
6. 보드는 *가능하면 좌우 대칭*.

이 중 #2/#3/#4는 코드 변경 없음. 진짜 작업: **#1(색 진행), #5(2색 큐), #6(대칭)**.

## 결정

### #1 색 진행 — 구간 내 랜덤 (Decision #1: Option B)
| 레벨 인덱스 | 색 수 |
|---|---|
| L1–10  | **2 또는 3** (랜덤) |
| L11–20 | 3 또는 4 |
| L21–30 | 4 또는 5 |
| L31–40 | 5 또는 6 |
| L41–50 | 6 또는 7 |
| L51+   | 6 또는 7 (고정) |

공식: `bucket = min(level_idx // 10, 4)`, `colors ~ Uniform{2 + bucket, 3 + bucket}`.

### #5 발사 슈터 — 2색 큐 (Decision #2: Option A)
- 매 턴 큐 `[current, alt]` 두 색을 보유.
- 봇/플레이어는 *둘 중 하나*를 골라 발사.
- 발사 후 그 슬롯만 새 색으로 보충.
- 룩어헤드 전략은 *남은 색 + 다음 랜덤*을 평균.

봇 인터페이스 변경:
```
choose_shot(board, queue: list[int], rng) -> (row, col, queue_index) | None
```

### #6 대칭 — 부드러운 좌우 대칭 (Decision #3: Option B)
- **Stage 1**: 시드를 좌측 절반에만 배치, 미러 페어로 우측에도 동시 배치 (같은 색).
  성장은 양쪽 독립으로 진행 (자연스러움).
- **Stage 2**: 진화 시 70% 확률로 짝 셀(`(r, c)` ↔ `(r, cols-1-c)`)을 같이 변형, 30%는 단독.

미러 함수: `mirror_col(c) = cols - 1 - c`.
홀수 행은 시각적으로 반 칸 어긋나지만 인덱스 미러는 동일 — 시각 대칭은 *근사*.

### 발사 수 보정
색이 늘면 매칭이 어려워지니 발사 수도 비례 증가.
`shots_remaining = 12 + 2 × (num_colors − 2)` → 2색=12, 7색=22.

## 근거

1. **공고 매칭**: 라이브 게임의 진짜 메커닉(2색 swap = Bubble Pop Origin)을 직접 채용 → 회사 IP와의 정합성 ↑.
2. **Iteration 서사**: ADR 0005의 발견을 *디자이너 피드백으로 진화시킨다*는 라이브 운영 패턴.
3. **다양성**: 색 수가 변동하므로 회귀의 통제변수로 `num_colors`/`level_index` 추가 → 진짜 효과를 분리.

## 트레이드오프

- **호환성 깨짐**: Iteration 1 봇 인터페이스/spec과 호환 안 됨. iter1 산출물은 `levels_iter1/`, `reports_iter1/`로 보존 → 비교 가능.
- **솔버 비용 ↑**: 봇이 두 색 후보를 모두 시뮬해야 함 → ~2배 느림. 100레벨 배치 ~45분 예상.
- **대칭 강제로 진화 자유도 ↓**: stage 2 수렴이 약간 느려짐. max_generations를 600으로 상향.

## 후속

- ADR 0005의 권고(`shot_pressure` 제거)는 *이 ADR과 동시에 적용*: 6차원 → 5차원 + `level_index`(통제변수, 회귀에만).
- Iteration 1과 2의 회귀 R²를 PORTFOLIO에서 비교한다.
